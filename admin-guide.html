01アクセス方法と合言葉

管理画面は一般公開ナビからは到達できません。以下の直URLを使用します。

環境URL
ローカル (PC)http://localhost:8001/projects/kodoco/admin.html
ローカル (スマホ実機)http://192.168.0.10:8001/projects/kodoco/admin.html
本番 (GitHub Pages)https://99letters.github.io/projects/kodoco/admin.html

初回アクセス時にゲート画面が出るので、合言葉を入力します。

🔑 合言葉

kodoco-admin-2025

認証は sessionStorage に保存されるため、タブを閉じるまで有効。別タブ/別端末では再入力が必要。

注意: これはクライアントサイドの簡易ゲートです。真のセキュリティではなく「URLを偶然踏んだ一般ユーザーに中身を見せない」目的の障壁。本当に秘匿したいデータがある場合はサーバー側認証が必要です。

03データ操作 (エクスポート/インポート)

ユーザーデータ(お気に入り・声・登録スポット・アプリ状態)を JSON でまとめて書き出し・取り込みできます。旧 settings → データ セクションから移設した機能。

エクスポート

ボタン押下で kodoco_backup_YYYY-MM-DD.json というファイルがダウンロードされます。構造:

{
  "format": "kodoco.backup",
  "version": 1,
  "exportedAt": "2026-04-24T08:00:00.000Z",
  "voices": [...],
  "userSpots": [...],
  "state": { "likes": [...], "children": [...], ... }
}

インポート

  • ファイル選択 → 確認ダイアログ → 既存データを上書き
  • format: "kodoco.backup" でないファイルは拒否
  • インポート直後に画面を再描画(再読み込みなし)
使いどころ: バグ報告用の再現データ固定化、PC移行、ブラウザ切替、テストデータの保存・再利用。

04localStorage ビューア

kodoco_* で始まる全キーを一覧表示。キーごとに サイズ / 値プレビュー / 削除ボタンを提供。

扱うキー一覧

キー内容
kodoco_stateお気に入り・お子さん情報・設定 (JSON)
kodoco_voices投稿された声/口コミ (JSON配列)
kodoco_user_spotsユーザー登録スポット (JSON配列)
kodoco_uid匿名ユーザーID (自動生成)
kodoco_mascot_pref選択中のお気に入りキャラ
kodoco_versionアプリバージョン(互換性管理用)
kodoco_admin_ok管理画面への認証状態(これは削除ボタン無効)

その他、キャッシュ用の kodoco_spot_rain_cache、kodoco_weather_cache、kodoco_weather_loc なども含まれる場合あり。

05スポットデータ確認

スポットの URL / 電話番号 などのデータ品質チェックは別の画面 check.html に分離されています。adminからはリンクで起動するだけ。

  • フィルタ: すべて / URL未設定 / 電話未設定 / 確認済み / 問題あり / 未確認
  • カテゴリ別絞り込み(公園 / 室内 / カフェ / 体験 / ご飯 / 犬連れ)
  • 問題ありリストをクリップボードに出力

06キャラ解禁テスト

最も実用的なセクション。 実データを作り込まずに、UI/報酬の解禁段階を即座に再現できます。

トラック

トラックカウント元解禁段階
💚 お気に入りstate.likes.length1件=マフラー / 5件=ハナちゃん / 10件=帽子 / 20件=王冠
💬 声の主kodoco_voices.length1件=拍手強化 / 5件=クロちゃん / 10件=レア口コミ / 20件=金マフラー
📍 スポット登録家kodoco_user_spots.length1件=感謝バッジ / 3件=船帽子 / 10件=開拓者

操作

  • 各ボタンを押すと対応する localStorage キーに仮データが書き込まれます
  • 「0件に戻す」で該当キーを空配列に上書き
  • 右上のダッシュボードも自動更新される
⚠ 実データを上書きします。 本当のテスト前に「エクスポート」で現在のデータを必ず退避させてください。

07キャッシュ / Service Worker 制御

ボタン動作使いどころ
🗑 CacheStorage をクリアcaches.delete() 全削除画像/JSの更新が反映されない
⚡ Service Worker を解除ServiceWorkerRegistration.unregister()古い SW が新しい更新を掴まない
💣 localStorage 全消去kodoco_* を全削除(認証キー除く)初回ユーザー体験の再現・テスト

08使いどころ (シナリオ例)

A

解禁キャラの見た目を確認したい

admin → キャラ解禁テスト → 「likes=5」を押す → app.html へ移動 → 設定タブの 🏆 あしあと を確認 → ハナちゃん解禁の見た目が出る

B

バグ報告用のデータを保存

バグを踏んだら即 admin → エクスポート → JSON保存 → 後で別環境でインポートして再現

C

初回オンボーディングを再テスト

admin → 「localStorage 全消去」 → 「SW解除」 → app.html を ⌘ Shift R でハードリロード → 初回ウィザードが出る

D

キャッシュが古くて新しいCSS が反映されない

admin → CacheStorage クリア + SW 解除 → ハードリロード

E

特定の voices だけ消したい

admin → localStorage ビューア → kodoco_voices の「削除」 → voicesだけ消える(likes等は残る)

09設定の変更方法

合言葉を変える

admin.html 内の以下の定数を編集:

const ADMIN_KEY = 'kodoco-admin-2025';   // ← 好きな文字列に

認証の保存期間を変える

デフォルトは sessionStorage (タブを閉じるまで)。永続化したい場合は submitGate() 内で localStorage.setItem(AUTH_KEY, '1') に変更可能。

表示するキー/統計を追加する

renderStats() の stats 配列に項目を追加すれば、自動的にダッシュボードに表示されます。

app.html 側の変更点

設定タブの「データ」セクション(JSONエクスポート/インポート)は display:none で非表示化。DOMは保持してあるので、将来復活させたい場合は style="display:none;..." を外すだけ。該当箇所は app.html で settings-section-title の「データ」コメントを検索してください。

11セキュリティについて

  • admin.html は GitHub Pages で公開されます。URL を知っていればアクセス可能。
  • 合言葉はソースコードにハードコードされているため、開発者ツールから覗けます。真のセキュリティではなく、障壁です。
  • 扱うデータは「このブラウザの localStorage」のみ。他ユーザーのデータには触れません。
  • robots メタタグで noindex,nofollow を設定済み(検索エンジン登録回避)。
  • 将来本格的なマルチユーザー化する場合は、Cloudflare Access・Basic認証・Firebase Auth 等への置き換えが必要。
